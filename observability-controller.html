Many AI workflows begin reasoning before they know whether a request contains enough information to produce a useful result.

The Observability Controller acts as a lightweight decision layer before downstream execution. It evaluates request sufficiency and routes requests to clarify or proceed before models, retrieval systems, tools or agents consume additional compute.

The controller is model agnostic and is designed to work alongside your existing AI stack. It can be integrated ahead of AI models, RAG systems, agent workflows and internal assistants through a simple API, with controlled evaluation before wider production rollout.

Data handling

The controller evaluates each request and returns a routing decision such as proceed or clarify.

The current beta API is stateless by design. It does not require conversation memory, stored prompt history or training on user data and does not intentionally retain prompts between requests.
